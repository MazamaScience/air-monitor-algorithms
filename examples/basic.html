air-monitor-algorithms – Browser Example

Purpose

The air-monitor-algorithms package provides scientific algorithms for analyzing hourly time series data, with a focus on air quality monitoring. It supports the air-monitor ecosystem, which works with air quality monitoring data archives hosted by the US Forest Service AirFire team.

All time series are assumed to be on a regular hourly axis with no gaps. Missing values are represented as null. All timestamps are handled as UTC values using the Luxon date-time library.

This page demonstrates three of the package's primary functions using a 14-day synthetic PM2.5 dataset.

Libraries Used in This Example

  • air-monitor-algorithms — provides the pm_nowcast, dailyStats, and diurnalStats functions used below.
  • Luxon — provides the DateTime class for creating and manipulating UTC timestamps. All timestamp inputs to air-monitor-algorithms must be Luxon DateTime objects in the UTC timezone.

Both libraries are loaded directly from a CDN — no installation or build step is required to run this page.

Example Dataset

The dataset is generated in-page and consists of 336 synthetic hourly PM2.5 values spanning 14 days (1–14 July 2024) in the America/Los_Angeles timezone. Values are constructed from three overlapping components that together mimic realistic air quality conditions during a wildfire smoke event:

  • Background baseline — a constant 10 µg/m³ representing typical clean-air conditions.
  • Diurnal cycle — a 24-hour sinusoid driven by daily human activity (traffic, heating, cooking). Concentrations peak around 9 AM and fall to their lowest around 9 PM, adding up to 16 µg/m³ above the baseline.
  • Episodic smoke event — a smooth bell curve spanning all 14 days, representing wildfire smoke drifting into and then clearing from a region. The event peaks at day 7 and adds up to ~80 µg/m³, bringing combined concentrations to roughly 105 µg/m³ at the peak.

NowCast

    ...
    const nowcast = pm_nowcast(pm25);

The pm_nowcast() function implements the US EPA NowCast algorithm which converts a sequence of hourly PM2.5 measurements into a weighted rolling average designed for communicating current air quality conditions to the public. Unlike a simple moving average, NowCast uses a 12-hour window and gives progressively more weight to the most recent hours — and responds especially fast when concentrations are changing rapidly, such as during the onset or clearing of a smoke event.

Daily Statistics

    ...
    const daily = dailyStats(datetime, pm25, timezone);

The dailyStats() function aggregates all 24 hourly observations for each calendar day into a single summary row. Day boundaries are determined using local midnight-to-midnight in the America/Los_Angeles timezone, even though all timestamps are stored in UTC.

Each row reports the count of valid (non-missing) hourly values alongside the daily minimum, mean, and maximum PM2.5. The count column is useful for data-quality screening: a day with fewer than 24 valid values may not be fully representative. The smoke event peak is visible in the rising mean and maximum values from day 4 through day 10.

Diurnal Profile

    ...
    const diurnalProfile = diurnalStats(datetime, pm25, timezone, 14);

The diurnalStats() function collapses all 14 days of hourly observations into a single "average day" by grouping values by hour-of-day (0–23) and computing statistics across all days. This reveals the characteristic daily shape of a pollutant signal and is useful for understanding when concentrations are typically highest or lowest over a multi-day period.

In this synthetic dataset the diurnal signal peaks around 09:00 and reaches its minimum around 21:00. Because the smoke event elevates all hours across the middle days, the min and max columns show a wide spread — the min reflects cleaner days at the start and end of the record, while the max reflects smoke-event days near the peak.
